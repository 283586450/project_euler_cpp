# Problem 0007: 10001st Prime

## Problem Summary

- Source: Project Euler #7
- Goal: find the 10001st prime number

## Main Idea

Generate primes incrementally and stop as soon as the target index is reached.

## Primary Solution

Keep a growing list of primes, test each odd candidate against the primes found so far, and stop once the list contains 10001 primes.

## Alternate Approaches

- A sieve works too, especially if the upper bound is known in advance.
- More advanced prime generators are unnecessary for this input size.

## Implementation Notes

- Trial division stops once `prime > candidate / prime`, which avoids overflow and extra work.
- Only odd candidates are tested after `2`.

## Complexity

- Time: subquadratic for this input size, with enough pruning to be practical
- Space: `O(n)` for the list of discovered primes

## Verification

- Known answer: `104743`
