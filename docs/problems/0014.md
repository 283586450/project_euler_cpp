# Problem 0014: Longest Collatz Sequence

## Problem Summary

- Source: Project Euler #14
- Goal: find the starting number under one million that produces the longest Collatz chain

## Main Idea

Memoize chain lengths so repeated subchains are only evaluated once.

## Primary Solution

For each starting value, follow the Collatz sequence until a cached length is reached, then backfill the lengths for the visited path.

## Alternate Approaches

- Pure recursion with memoization.
- Iterative brute force without caching, which is much slower.

## Implementation Notes

- The implementation stores lengths only for values below the limit.
- A temporary path buffer makes the backfill step simple and safe.

## Complexity

- Time: much better than naive brute force, with each in-range value cached once
- Space: `O(n)` for the memo table

## Verification

- Known answer: `837799`
